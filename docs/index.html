godnote

ワークスペースで動くローカルファーストノートアプリ

`godnote` は、ユーザーが選んだフォルダをそのままデータの正本にするノートアプリです。 デスクトップ版で編集し、GitHub Pages は説明と使い方のドキュメントサイトとして公開します。

はじめかた 考え方 今後の方針

何をするアプリか

  • ワークスペースを作成して、そこにノートデータを保存する
  • ワークスペースを開いて、既存データを編集する
  • 画像や PDF を PNG に変換してノートに保存する
  • 将来的には Electron 化して OS ごとの standalone アプリにする

GitHub Pages の役割

  • アプリ本体は置かない
  • 使い方と考え方を説明する
  • 開発中の前提や今後の方針をまとめる
  • デスクトップ版への導線を案内する

はじめかた

  1. デスクトップ版 `godnote` を起動する
  2. Home で「ワークスペースを作成」または「ワークスペースを開く」を選ぶ
  3. 作った workspace の中に `manifest.json`、`subjects/`、`notes/` ができる
  4. ノートを編集すると、画像は workspace 内の `assets/images/` に保存される

ワークスペースの構成

<workspace>/
├── manifest.json
├── subjects/
├── notes/
└── .github/workflows/deploy-pages.yml

現在の方針

  • Git 管理はユーザーに任せる
  • GitHub Pages は説明サイトにする
  • sample data は最小限に保つ
  • 次の大きな工程は Electron 化

今後の方針

  1. デスクトップ版のワークスペース編集を安定させる
  2. GitHub Pages の説明サイトを整える
  3. Electron に移して standalone 配布へ進める
  4. OS ごとのビルドを整える
